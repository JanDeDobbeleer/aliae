By default, aliae expects a `YAML` file called `.aliae.yaml` in the user `HOME` folder.
You can however also specify a custom location using the `--config` flag on initialization.

```bash
eval "$(aliae init zsh --config '/Users/aliae/configs/aliae.yaml')"
```

The custom location can be a local file, or a URL pointing to a remote config.

## Layout

All shared aliases are defined using the `aliae` property:

```yaml
aliae:
  - alias: a
    value: aliae
  - alias: hello-world
    value: echo "hello world"
    type: function
```

## Aliae

### Alias

| Name    | Type     | Description                                                                                                                                       |
| ------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| `alias` | `string` | the alias name                                                                                                                                    |
| `value` | `string` | the command(s) you want to execute                                                                                                                |
| `type`  | `string` | <ul><li>`command`: a regular alias, value is a one-liner (**default**) </li><li>`function`: a code block to be placed inside a function</li></ul> |
